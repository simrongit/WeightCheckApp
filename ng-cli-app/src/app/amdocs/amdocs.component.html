Fallout Trend Analyzer

May 2016
Idea behind this tool was to give concise information which can help us in understanding fallout trend thus helping in understanding pattern which cause fallout. That will lead to root cause
  • Quick Scan
    1. It offers no delay in seeing graphs for various activity. Data is cached on it is first launched.
    2. You can switch sample size to see how exactly it behaves (15min to 6hrs)
    3. Rest button to load data again
  • Detail Scan
    1. It provides various filters which help in analysis. For e.g. you can find how many Change Mass orders got failed and successfully passed.
    2. It caches result for each activity so jumping back is quick. But first load is slower as it pulls detailed information form DB.
    3. It shows multiple lines if you want to see comparison between type. For e.g. failed orders vs non failed.

You need to create MnP in a format so that tool can use it. Format standard used is JSON. So you can read in depth about square braces, double quote, commas etc. Anything which you see on left side of colon(:) is static. No change allowed It starts with what input you need to user. Like I used order_action_id, you can give any name. Its recommended to write MnP in lower case in order to avoid accidental string match between query and replaced input parameter Then you write select query to validate input. Same is bound with user friendly message which you want to show to user. NOTE : from select query only 1 row is expected. Select items can only be String.
